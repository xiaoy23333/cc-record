# CLAUDE.md — 唱片销售管理系统

## 项目概述

北方工业大学学术工程实践（计算机）课程设计 —— 命令行唱片销售管理系统。使用 C++ 语言，控制台交互，通过结构体 + 文件存储数据。

## 技术栈

- **语言**：C++（g++ 编译）
- **平台**：Windows，MinGW64
- **数据存储**：结构体 + 二进制/文本文件 `.dat`
- **交互方式**：纯控制台命令行菜单

### 命名规范

| 类别 | 风格 | 示例 |
|------|------|------|
| 结构体 | PascalCase | `Record`, `SaleInfo` |

- **函数定义**：左大括号换行

```cpp
void addRecord()
{
    // body
}

if(condition)
{
    // body
}
else
{
    // body
}

for(int i = 0; i < n; i++)

    // body
}
```

### 缩进 & 空格

- **缩进**：Tab
- `for`/`if`/`while` 后不加空格：`if(condition)`、`for(int i = 0;...`
- `>>` 两侧加空格：`cin >> n;`
- 嵌套模板 `>` 之间加空格：`vector<vector<int> >`

## 数据结构设计

### 四大结构体

1. **Record（唱片基本信息）**：唱片编号、唱片名、歌手名、风格、价格、发行日期、备注
2. **InputRecord（进货信息）**：唱片编号、唱片名称、进货数量、进货日期、进货人
3. **StockRecord（库存信息）**：唱片编号、唱片名称、库存数量、进货数量、销售数量
4. **SellRecord（销售信息）**：销售编号、唱片编号、唱片名称、销售价格、销售数量、销售金额、客户名称、销售日期

### 数据文件

| 文件 | 存储内容 |
|------|----------|
| `pwd.dat` | 系统密码 |
| `record_base.dat` | 唱片基本信息 |
| `record_input.dat` | 进货记录 |
| `record_stock.dat` | 库存信息 |
| `record_sell.dat` | 销售记录 |

## 开发要求

1. 画出功能模块图
2. 整个程序不能只有 `main` 函数，必须按功能模块编写函数，功能函数 ≥10 个
3. 唱片基本信息、进货信息、库存信息、销售信息必须以结构体形式存储
4. 以上四类信息必须存储在各自文件中
5. 通过链表 + 文件结合，至少完成 3 个基本功能

## 禁止事项

1. ❌ 不要过度边界检查和特判 — 代码保持简洁，像大一学生写的风格
2. ❌ 不要引入第三方库或复杂设计模式
3. ❌ 不要用 C++ STL 容器替代文件存储（文件存储是评分要求）
4. ❌ 不要混用 `printf`/`scanf` 和 `cout`/`cin` — 统一用 `cout`/`cin`
5. ❌ 不要用 `NULL` — 用 `0`
6. ❌ 不要在 `for`/`if` 等控制结构后省略 `{}`
7. ❌ 不要用 `true`/`false`，用 `1`/`0`

## 开发方式

由于最后需要向他人讲解代码，必须理解每处代码的功能和含义。因此采用**分模块逐步开发**：

- 每次只生成一个功能模块的代码，不要一口气全部生成
- 每个模块生成前先讲解该模块的设计思路和数据结构
- 代码中保留中文注释说明关键逻辑
- 生成后解释代码执行流程，确保每行都理解到位
- 模块开发顺序按功能编号（1→2→3→...→10，最后做扩展功能）
